## 三、数据库模型（Prisma）

### 3.1 AudioFile（音频文件）

```prisma
model AudioFile {
  id            String   @id @default(cuid())
  fileName      String
  fileSize      Int
  duration      Int?     // 秒（转换后获取准确值）
  format        String
  originalPath  String   // 原始文件路径
  filePath      String   // 转换后的 WAV 路径
  status        String   @default("pending") // pending, converting, ready, transcribing, completed, failed
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  transcript Transcript?
}
```

### 3.2 Task（任务状态）

```prisma
model Task {
  id         String   @id @default(cuid())
  type       String   // "transcribe" | "generate"
  status     String   @default("pending") // pending, processing, completed, failed
  progress   Int      @default(0) // 0-100
  result     String?  // JSON 结果
  error      String?  // 错误信息
  createdAt  DateTime @default(now())
  updatedAt  DateTime @updatedAt
}
```

## 四、API 实现

### 4.1 POST /api/upload - 上传音频

**请求**：multipart/form-data，字段 `file`

**响应**：
```json
{
  "audioId": "cuidxxx",
  "fileName": "podcast.mp3",
  "duration": 1800,
  "status": "pending"
}
```

**实现要点**：
- 使用 `formidable` 解析 multipart 文件
- 保存到 `/uploads/{audioId}/original.{ext}`
- 转换后使用 `ffprobe` 获取准确时长
- 原始文件保存在 `originalPath`，转换后的 WAV 保存在 `filePath`

### 4.2 POST /api/transcribe - 提交转录

**请求**：
```json
{ "audioId": "cuidxxx" }
```

**响应**：
```json
{ "taskId": "task_xxx" }
```

**实现要点**：
- 创建 Task 记录，type = "transcribe"
- 调用 whisper.cpp 本地转录（见 5.1）
- 异步处理，返回 taskId 供轮询

### 4.3 GET /api/task/[taskId] - 查询状态

**响应**：
```json
{
  "status": "processing",
  "progress": 45,
  "result": null
}
```

### 4.4 POST /api/generate - 生成文章

**请求**：
```json
{ "transcriptId": "transcript_xxx" }
```

**响应**：
```json
{
  "article": {
    "id": "article_xxx",
    "title": "播客标题",
    "content": "# 播客标题\n\n..."
  },
  "extracted": {
    "tags": ["标签1", "标签2"],
    "highlights": [...],
    "summary": "..."
  }
}
```

**实现要点**：
- 调用 Minimax API（见 5.2）
- 生成 Markdown + 提取 tags/highlights

### 4.5 GET /api/export/[id] - 导出 Markdown

**响应**：Markdown 文本，Content-Type: text/markdown

## 五、外部 API 集成

### 5.1 音频格式转换（fluent-ffmpeg）

```typescript
// src/lib/audio-converter.ts
import { convertToWav, getAudioInfo, isFfmpegInstalled } from './audio-converter';

// 检测 ffmpeg 是否安装
const installed = await isFfmpegInstalled();
if (!installed) {
  throw new Error('FFmpeg is not installed');
}

// 获取音频信息
const info = await getAudioInfo('/path/to/audio.mp3');
// info = { duration: 1800, format: 'mp3' }

// 转换为 WAV（16kHz, 16-bit, 单声道）
await convertToWav('/path/to/audio.mp3', '/path/to/output.wav');
```

**转换参数**：
- 采样率：16000 Hz
- 位深：16-bit PCM
- 声道：单声道（mono）

**依赖安装**：
```bash
# 系统需要安装 ffmpeg
# Windows: winget install Gyan.FFmpeg
# macOS: brew install ffmpeg
# Ubuntu: sudo apt install ffmpeg
```

### 5.2 whisper.cpp 本地转录

```typescript
// src/lib/whisper.ts
import { init, transcribe } from './whisper';

// 初始化（只需一次）
await init({
  model: 'whisper.cpp/models/ggml-large.bin',
  cuda: true,
});

// 转录音频
const result = await transcribe('/path/to/audio.wav', 'zh');
// result = { language: 'zh', fullText: '...', segments: [...] }
```

**构建步骤**：
```bash
# 1. 编译 whisper.cpp（需 CMake + CUDA Toolkit）
powershell ./scripts/build-whisper.ps1

# 2. 编译 Node.js 原生插件
npm run build:addon

# 3. 下载模型（脚本会自动下载）
# 模型保存到: whisper.cpp/models/ggml-large.bin
```

**环境变量**：
```bash
WHISPER_MODEL_PATH=whisper.cpp/models/ggml-large.bin
WHISPER_USE_CUDA=1  # 0 = CPU only
```

### 5.3 Minimax

```typescript
// src/lib/minimax.ts
import { env } from '$env';

export async function generateArticle(transcript: string): Promise<{
  title: string;
  content: string;
  tags: string[];
  highlights: Highlight[];
  summary: string;
}> {
  const response = await fetch(`${env.MINIMAX_API_BASE}/chat/completions`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${env.MINIMAX_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      model: 'minimax2.7',
      messages: [
        {
          role: 'system',
          content: `你是一个播客文字整理助手。请将转录文本整理成结构化Markdown...
（完整提示词见附录 A）`
        },
        {
          role: 'user',
          content: transcript
        }
      ]
    })
  });

  return parseResponse(response);
}
```

## 八、附录

### 附录 A：Minimax 提示词

```
你是一个播客文字整理助手。请将以下转录文本整理成结构化Markdown文章。

要求：
1. 生成一个简洁准确的标题
2. 生成目录（## 大纲），包含2-5个主要章节
3. 按逻辑章节组织内容，每个章节用 ### 标记
4. 保留关键引述和金句，用 > 引用样式
5. 生成5-10个主题标签
6. 提取3-5个高亮金句（观点鲜明，50字以内）
7. 生成200字以内的摘要

输出格式（JSON）：
{
  "title": "标题",
  "outline": ["章节1", "章节2"],
  "content": "完整Markdown内容",
  "tags": ["标签1", "标签2"],
  "highlights": [{"text": "金句", "context": "上下文"}],
  "summary": "摘要"
}
```

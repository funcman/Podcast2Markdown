# Podcast2Markdown 部署指南

## 一、本地开发

### 1.1 环境要求

- Node.js v22.18.0
- Git
- C/C++ 编译工具：
  - **Windows**: Visual Studio 2022 + CMake + Ninja
  - **Linux**: GCC/Clang + CMake + Make

**可选（GPU 加速）：**
- NVIDIA GPU + CUDA Toolkit 11.x

### 1.2 安装依赖

```bash
cd Podcast2Markdown
npm install
```

### 1.3 编译 whisper.cpp（语音转录引擎）

#### Windows (PowerShell) - 推荐使用脚本

```powershell
# 使用构建脚本（自动处理所有步骤）
powershell ./scripts/build-whisper.ps1

# 或指定模型大小
powershell ./scripts/build-whisper.ps1 small   # 小模型
powershell ./scripts/build-whisper.ps1 base   # 基础模型
powershell ./scripts/build-whisper.ps1 -CPU  # CPU 版本
```

#### Windows 手动构建

如果脚本失败，按以下步骤手动构建：

```powershell
# 1. 克隆 whisper.cpp（需要完整克隆，不能用 --depth 1）
git clone --branch v1.7.1 https://github.com/ggerganov/whisper.cpp.git

# 2. 创建构建目录
cd whisper.cpp
mkdir build
cd build

# 3. 配置（CPU 版本，务必加 /utf-8 处理中文）
cmake .. -G Ninja -DCMAKE_C_FLAGS="/utf-8" -DCMAKE_CXX_FLAGS="/utf-8"

# 4. 编译
ninja
```

**CUDA 版本**（需要 CUDA Toolkit + nvcc）：
```powershell
cmake .. -G Ninja -DGGML_CUDA=ON -DCMAKE_C_FLAGS="/utf-8" -DCMAKE_CXX_FLAGS="/utf-8"
ninja
```

#### Linux/macOS

```bash
cd whisper.cpp
mkdir build && cd build

# CPU 版本
cmake .. -DCMAKE_C_FLAGS="-O3" -DCMAKE_CXX_FLAGS="-O3"
make -j$(nproc)

# CUDA 版本
cmake .. -DGGML_CUDA=ON -DCMAKE_C_FLAGS="-O3" -DCMAKE_CXX_FLAGS="-O3"
make -j$(nproc)
```

**注意：**
- whisper.cpp 源码包含中文字符，Windows MSVC 编译必须加 `/utf-8`
- 首次构建需要下载模型（约 1-3GB）

### 1.4 下载 Whisper 模型

模型文件需放置在 `whisper.cpp/models/` 目录：

| 模型 | 大小 | 推荐场景 |
|------|------|----------|
| ggml-base.bin | 142 MB | 测试/低配设备 |
| ggml-small.bin | 466 MB | 日常使用 |
| ggml-medium.bin | 1.5 GB | 高质量需求 |
| ggml-large.bin | 2.9 GB | 最高质量 |

**Windows PowerShell：**
```powershell
$model = "base"  # 替换为需要的模型: base, small, medium, large
$modelFile = "ggml-${model}.bin"
$modelsDir = "D:\Projects\Podcast2Markdown\whisper.cpp\models"
$url = "https://huggingface.co/ggerganov/whisper.cpp/resolve/main/models/$modelFile"

New-Item -ItemType Directory -Force -Path $modelsDir | Out-Null
Invoke-WebRequest -Uri $url -OutFile "$modelsDir\$modelFile" -UseBasicParsing
```

**Linux/macOS：**
```bash
cd whisper.cpp/models
bash ./download-ggml-model.sh large
```

**或使用 curl：**
```bash
curl -L "https://huggingface.co/ggerganov/whisper.cpp/resolve/main/models/ggml-large.bin" -o ggml-large.bin
```

**验证模型：**
```bash
ls -lh whisper.cpp/models/ggml-base.bin
# base 应显示约 142MB，large 应显示约 2.9GB
```

### 1.5 配置环境变量

创建或编辑 `.env` 文件：

```bash
# Whisper 本地转录配置
WHISPER_MODEL_PATH=whisper.cpp/models/ggml-large.bin
WHISPER_USE_CUDA=1

# MINIMAX API（文章生成）
MINIMAX_API_KEY=your_minimax_api_key
MINIMAX_API_BASE=https://api.minimaxi.com/v1

# 数据库
DATABASE_URL="file:./dev.db"
```

### 1.6 初始化数据库

```bash
npx prisma generate
npx prisma db push
```

### 1.7 启动开发服务器

```bash
npm run dev
```

访问 http://localhost:3000

### 1.8 测试转录功能

```bash
# 使用 whisper.cpp 自带测试音频
cd whisper.cpp
./build/bin/main.exe -m models/ggml-large.bin -f samples/jfk.wav
```

## 三、常见问题

### whisper.cpp 编译报错 "文本后缀"

Windows MSVC 编译时出现中文乱码错误：
```
error C3688: 文本后缀"銆"无效
```

**解决方案：** 配置 CMake 时添加 `/utf-8` 标志：
```powershell
cmake .. -G Ninja -DCMAKE_C_FLAGS="/utf-8" -DCMAKE_CXX_FLAGS="/utf-8"
```

### 模型下载失败

```bash
# 检查网络
curl -I https://huggingface.co

# 使用浏览器下载后手动放置到 whisper.cpp/models/ 目录
# 或使用国内镜像
```

### nvcc 未找到

CUDA 编译需要安装 CUDA Toolkit 并确保 `nvcc` 在 PATH 中：
```powershell
# 检查 nvcc
nvcc --version

# 如果未安装，只能使用 CPU 版本
powershell ./scripts/build-whisper.ps1 -CPU
```

## 四、环境变量汇总

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `WHISPER_MODEL_PATH` | Whisper 模型路径 | `whisper.cpp/models/ggml-large.bin` |
| `WHISPER_USE_CUDA` | 是否使用 GPU | `1`（自动检测） |
| `MINIMAX_API_KEY` | Minimax API Key | - |
| `MINIMAX_API_BASE` | Minimax API 地址 | `https://api.minimaxi.com/v1` |
| `DATABASE_URL` | SQLite 数据库路径 | `file:./dev.db` |

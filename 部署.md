# Podcast2Markdown 部署指南

## 一、本地开发

### 1.1 环境要求

- Node.js 18+ （建议 20+）
- Git
- C/C++ 编译工具：
  - **Windows**: Visual Studio 2022 + CMake + Ninja（可选）
  - **Linux**: GCC/Clang + CMake + Make

**可选（GPU 加速）：**
- NVIDIA GPU + CUDA Toolkit 11.x 或更高

### 1.2 安装依赖

```bash
cd Podcast2Markdown
npm install
```

### 1.3 编译 whisper.cpp（语音转录引擎）

#### Windows (PowerShell) - 推荐使用脚本

```powershell
# 使用构建脚本（自动处理所有步骤）
powershell ./scripts/build-whisper.ps1

# 指定模型大小
powershell ./scripts/build-whisper.ps1 small   # 小模型（466MB）
powershell ./scripts/build-whisper.ps1 base   # 基础模型（142MB）
powershell ./scripts/build-whisper.ps1 large  # 大模型（3.1GB）

# CPU 版本（无 GPU）
powershell ./scripts/build-whisper.ps1 -CPU

# 指定 GPU 架构（RTX 3060 = 86）
powershell ./scripts/build-whisper.ps1 large -GPUArch 86
```

#### Windows 手动构建

如果脚本失败，按以下步骤手动构建：

```powershell
# 1. 克隆 whisper.cpp（需要完整克隆，不能用 --depth 1）
git clone --branch v1.7.1 https://github.com/ggerganov/whisper.cpp.git

# 2. 创建构建目录
cd whisper.cpp
mkdir build
cd build

# 3. 配置（CPU 版本，务必加 /utf-8 和 /Zc:preprocessor 处理中文）
cmake .. -G Ninja -DCMAKE_C_FLAGS="/utf-8 /Zc:preprocessor" -DCMAKE_CXX_FLAGS="/utf-8 /Zc:preprocessor"

# 4. 编译
ninja
```

**CUDA 版本**（需要 CUDA Toolkit + nvcc）：
```powershell
# 设置环境变量（CUDA 13.2+ 需要）
$env:CCCL_IGNORE_MSVC_TRADITIONAL_PREPROCESSOR_WARNING = "1"

# 配置
cmake .. -G Ninja `
  -DGGML_CUDA=ON `
  -DCMAKE_C_FLAGS="/utf-8 /Zc:preprocessor" `
  -DCMAKE_CXX_FLAGS="/utf-8 /Zc:preprocessor /DCCCL_IGNORE_MSVC_TRADITIONAL_PREPROCESSOR_WARNING=1" `
  -DCMAKE_CUDA_FLAGS="-Xcompiler=/Zc:preprocessor" `
  -DCMAKE_CUDA_STANDARD=17

# 编译
ninja
```

#### Linux/macOS

```bash
cd whisper.cpp
mkdir build && cd build

# CPU 版本
cmake .. -DCMAKE_C_FLAGS="-O3" -DCMAKE_CXX_FLAGS="-O3"
make -j$(nproc)

# CUDA 版本
cmake .. -DGGML_CUDA=ON -DCMAKE_C_FLAGS="-O3" -DCMAKE_CXX_FLAGS="-O3"
make -j$(nproc)
```

**注意：**
- whisper.cpp 源码包含中文字符，Windows MSVC 编译必须加 `/utf-8`
- 首次构建需要下载模型（约 1-3GB）

### 1.4 下载 Whisper 模型

模型文件需放置在 `whisper.cpp/models/` 目录：

| 模型 | 大小 | 显存需求 | 推荐场景 |
|------|------|----------|----------|
| ggml-base.bin | 142 MB | 1GB | 测试/低配设备 |
| ggml-small.bin | 466 MB | 2GB | 日常使用 |
| ggml-medium.bin | 1.5 GB | 4GB | 高质量需求 |
| ggml-large-v3.bin | 3.1 GB | 8GB | 最高质量 |

**自动下载（通过构建脚本）**:
构建脚本会自动尝试下载指定模型，支持多种方式：
1. Invoke-WebRequest（默认）
2. BITS（备用）
3. curl.exe（备用）

**手动下载**:

Windows PowerShell：
```powershell
$model = "large-v3"  # 可选: base, small, medium, large-v3
$modelFile = "ggml-${model}.bin"
$modelsDir = "whisper.cpp\models"
$url = "https://huggingface.co/ggerganov/whisper.cpp/resolve/main/$modelFile"

New-Item -ItemType Directory -Force -Path $modelsDir | Out-Null

# 方式1: Invoke-WebRequest
try {
    Invoke-WebRequest -Uri $url -OutFile "$modelsDir\$modelFile" -UseBasicParsing
} catch {
    # 方式2: BITS
    Import-Module BitsTransfer
    Start-BitsTransfer -Source $url -Destination "$modelsDir\$modelFile"
}
```

**Linux/macOS**：
```bash
cd whisper.cpp/models
bash ./download-ggml-model.sh large-v3
```

**或使用 curl**：
```bash
curl -L "https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-large-v3.bin" -o whisper.cpp/models/ggml-large-v3.bin
```

**验证模型完整性**：
```bash
# Windows
ls -lh whisper.cpp/models/*.bin
# large-v3 应显示约 3.1GB，base 应显示约 142MB

# 如果文件大小不对，说明下载不完整，需要重新下载
```

### 1.5 配置环境变量

创建 `.env` 文件：

```bash
# Whisper 本地转录配置
WHISPER_MODEL_PATH=whisper.cpp/models/ggml-large-v3.bin
WHISPER_USE_CUDA=1

# MINIMAX API（文章生成）
MINIMAX_API_KEY=your_minimax_api_key
MINIMAX_API_BASE=https://api.minimaxi.com/v1

# 数据库
DATABASE_URL="file:./dev.db"
```

### 1.6 初始化数据库

```bash
npx prisma generate
npx prisma db push
```

### 1.7 编译 Node.js 原生插件

```bash
# 安装构建工具（node-gyp）
npm install

# 编译原生插件
npm run build:addon

# 调试版本
npm run build:addon:debug

# 重新构建
npm run rebuild:addon
```

## 二、FFmpeg 安装

本项目依赖 FFmpeg 进行音频格式转换。

### Windows

```powershell
winget install Gyan.FFmpeg
```

### macOS

```bash
brew install ffmpeg
```

### Ubuntu

```bash
sudo apt update
sudo apt install ffmpeg
```

### 验证安装

```bash
ffmpeg -version
```

如果显示版本信息，说明安装成功。

## 五、常见问题

### whisper.cpp 编译报错 "文本后缀"

Windows MSVC 编译时出现中文乱码错误：
```
error C3688: 文本后缀"銆"无效
```

**解决方案：** 配置 CMake 时添加 `/utf-8` 标志：
```powershell
cmake .. -G Ninja -DCMAKE_C_FLAGS="/utf-8" -DCMAKE_CXX_FLAGS="/utf-8"
```

### 模型下载失败

```bash
# 检查网络
curl -I https://huggingface.co

# 使用浏览器下载后手动放置到 whisper.cpp/models/ 目录
# 或使用国内镜像
```

### nvcc 未找到

CUDA 编译需要安装 CUDA Toolkit 并确保 `nvcc` 在 PATH 中：
```powershell
# 检查 nvcc
nvcc --version

# 如果未安装，只能使用 CPU 版本
powershell ./scripts/build-whisper.ps1 -CPU
```

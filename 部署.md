# Podcast2Markdown 部署指南

## 一、本地开发

### 1.1 环境要求

- Node.js v22.18.0
- npm

### 1.2 安装依赖

```bash
cd Podcast2Markdown
npm install
```

### 1.3 配置环境变量

创建 `.env` 文件：

```bash
# 灵芽 Whisper API
WHISPER_API_KEY=your_whisper_api_key
WHISPER_API_BASE=https://api.lingyaai.cn/v1

# MINIMAX API
MINIMAX_API_KEY=your_minimax_api_key
MINIMAX_API_BASE=https://api.minimaxi.com/v1

# 数据库
DATABASE_URL="file:./dev.db"
```

### 1.4 初始化数据库

```bash
# 生成 Prisma Client
npx prisma generate

# 创建数据库表
npx prisma db push
```

### 1.5 启动开发服务器

```bash
npm run dev
```

访问 http://localhost:3000

---

## 二、生产构建

### 2.1 本地构建

```bash
npm run build
npm start
```

### 2.2 Vercel 部署

1. 安装 Vercel CLI：
   ```bash
   npm i -g vercel
   ```

2. 登录 Vercel：
   ```bash
   vercel login
   ```

3. 部署：
   ```bash
   vercel
   ```

4. 生产环境部署：
   ```bash
   vercel --prod
   ```

### 2.3 环境变量配置（Vercel）

在 Vercel 项目设置中添加以下环境变量：

- `WHISPER_API_KEY`
- `WHISPER_API_BASE`
- `MINIMAX_API_KEY`
- `MINIMAX_API_BASE`
- `DATABASE_URL`（如使用云数据库）

## 五、故障排查

### 数据库连接失败

```bash
# 确认 .env 中 DATABASE_URL 正确
cat .env | grep DATABASE_URL

# 重新生成 Prisma Client
npx prisma generate
npx prisma db push
```

### 依赖安装失败

```bash
# 清除缓存重新安装
rm -rf node_modules package-lock.json
npm install
```

### API 调用失败

检查 `.env` 中的 API Key 和 Base URL 是否正确。

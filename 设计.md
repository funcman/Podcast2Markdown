# Podcast2Markdown 设计文档

## 一、项目概述

**Podcast2Markdown** 是一款将播客音频智能转换为结构化文章的工具。用户输入播客链接或关键词搜索，平台自动完成音频转文字、文字整理成大纲文章、提取关键词和金句，最终输出 Markdown 格式的文档。

### 目标用户
- 播客听众：收藏和整理喜欢的播客内容
- 内容创作者：快速将播客内容二次创作为文章
- 学习者：将音频内容转化为可检索的笔记

---

## 二、核心功能设计

### 2.1 本地音频上传

**功能描述**：用户直接上传本地音频文件（MP3、WAV、M4A 等格式），系统自动转换为 WAV 格式。

**实现方案**：
1. 前端支持选择文件上传（MP3、WAV、M4A 等格式）
2. 后端接收文件并存储到临时目录
3. **音频格式转换**：使用 fluent-ffmpeg 调用系统 ffmpeg 将非 WAV 格式转换为 WAV
   - 输出格式：WAV (16kHz, 16-bit, 单声道)
   - 这是 whisper.cpp 推荐的标准格式
4. 根据文件大小估算音频时长
5. 创建 AudioFile 记录，状态为 pending
6. 调用转录服务

**数据模型**：
```prisma
model AudioFile {
  id        String   @id @default(cuid())
  fileName  String
  fileSize  Int      // bytes
  duration  Int      // 秒（估算）
  format    String
  filePath  String   // 存储路径
  status    String   @default("pending")
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  transcript Transcript?
}
```

**状态流转**：
```
pending -> transcribing -> completed/failed
```

### 2.2 音频格式转换

**功能描述**：将用户上传的各种音频格式（MP3、M4A、OGG、FLAC 等）转换为 whisper.cpp 标准的 WAV 格式。

**技术选型**（已更新）：
- **方案**：fluent-ffmpeg（Node.js 封装，调用系统 ffmpeg）
- **依赖**：需要系统安装 ffmpeg

**转换参数**：
- 采样率：16000 Hz（whisper 推荐）
- 位深：16-bit PCM
- 声道：单声道（mono）

**实现方案**（已更新）：
1. 检查上传文件格式
2. 如果是 WAV 格式且参数符合要求，直接传递
3. 其他格式（MP3、M4A、OGG、FLAC），使用 **fluent-ffmpeg** 调用系统 ffmpeg 转换
4. 输出格式：WAV（16kHz, 16-bit, 单声道）
5. 原始文件保存到 `uploads/{audioId}/original.{ext}`
6. 转换后的文件保存到 `uploads/{audioId}/audio.wav`
7. 更新 AudioFile.filePath 指向转换后的文件
8. 使用 ffprobe 获取准确时长

**状态流转**（已更新）：
```
pending -> converting -> ready -> transcribing -> completed/failed
```

- **pending**: 文件上传完成，等待转换
- **converting**: 正在转换为 WAV
- **ready**: 转换完成，等待转录
- **transcribing**: 正在 Whisper 转录
- **completed**: 转录和文章生成完成
- **failed**: 转换或转录失败

**数据模型更新**（已更新）：
```prisma
model AudioFile {
  id             String   @id @default(cuid())
  fileName       String   // 原始文件名
  fileSize       Int      // 原始文件大小
  originalPath   String   // 原始文件路径
  filePath       String   // 转换后的 WAV 路径（兼容原有字段名）
  duration       Int?     // 秒（转换后获取准确值）
  format         String   // 原始格式
  status         String   @default("pending") // pending, converting, ready, transcribing, completed, failed
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt

  transcript Transcript?
}
```

### 2.3 音频转录

**功能描述**：将播客音频转换为文字。

**实现方案**：
1. 使用 **whisper.cpp** 本地推理
2. 通过 Node.js 原生插件（N-API）直接调用
3. 支持 **CUDA GPU 加速**（NVIDIA 显卡）
4. 返回带时间戳的字幕数据

**技术选型**：
- whisper.cpp v1.7.1：C++ 实现的高效 Whisper 推理，支持 CUDA
- Node.js 原生插件（binding.gyp + node-addon-api）：直接在 Node 进程中调用 whisper.cpp
- 模型：ggml-large-v3.bin（推荐）或 ggml-base.bin（低配）

**构建步骤**：
1. 运行 `npm run whisper:build` 编译 whisper.cpp（需 CMake + CUDA Toolkit）
2. 运行 `npm run build:addon` 编译 Node.js 原生插件

**成本优化策略**：
- 完全本地化，无 API 调用费用
- CUDA 加速大幅提升转录速度
- 支持 GPU 批量处理

**数据模型**：
```prisma
model Transcript {
  id          String   @id @default(cuid())
  audioFileId String   @unique
  audioFile   AudioFile @relation(fields: [audioFileId], references: [id])
  language    String
  fullText    String   // 完整转录文本
  segments    String   // JSON: TranscriptSegment[]
  status      String   @default("pending")
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  article Article?
}
```

```typescript
interface TranscriptSegment {
  start: number;  // 开始时间（秒）
  end: number;    // 结束时间（秒）
  text: string;   // 文本内容
}
```

### 2.3 文字整理与结构化

**功能描述**：将转录文字整理成带大纲和结构的 Markdown 文章。

**实现方案**：
1. 调用 **Minimax（MiniMax-M2.7）** 进行内容理解
2. 提示词设计：
   - 识别主题和核心观点
   - 按逻辑章节组织内容
   - 生成标题层级（H1/H2/H3）
   - 保留关键引述和金句
   - 提取主题标签和高亮金句
   - 生成 200 字以内摘要

**输出格式**：
```json
{
  "title": "文章标题",
  "outline": ["章节1", "章节2"],
  "content": "# 文章标题\n\n## 大纲...",
  "tags": ["标签1", "标签2"],
  "highlights": [{"text": "金句", "context": "上下文"}],
  "summary": "摘要"
}
```

**数据模型**：
```prisma
model Article {
  id           String     @id @default(cuid())
  transcriptId String     @unique
  transcript   Transcript @relation(fields: [transcriptId], references: [id])
  title        String
  content      String     // Markdown 内容
  summary      String?    // 摘要
  tags         String?    // JSON: string[]
  highlights   String?    // JSON: Highlight[]
  createdAt    DateTime   @default(now())
  updatedAt    DateTime   @updatedAt
}
```

### 2.4 任务状态管理

**功能描述**：跟踪转录和生成任务的进度。

**实现方案**：
1. 创建 Task 记录跟踪异步任务
2. 转录和文章生成在同一个异步流程中完成
3. 前端轮询任务状态

**数据模型**：
```prisma
model Task {
  id        String   @id @default(cuid())
  type      String   // "transcribe"
  status    String   @default("pending") // pending, processing, completed, failed
  progress  Int      @default(0)         // 0-100
  audioId   String?  // 关联的音频文件
  result    String?  // JSON 结果（包含 article 和 extracted）
  error     String?  // 错误信息
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

**进度划分**：
- 10%：开始处理
- 30%：开始 Whisper 转录
- 60%：转录完成，开始生成文章
- 100%：完成
```

### 2.5 关键词与金句提取

**功能描述**：从文章中提取 Tags 和高亮观点。

**实现方案**：
1. **Tags 提取**：使用 Minimax 分析主题标签（5-10个）
2. **金句提取**：识别观点鲜明、易于传播的短句（50字以内）
3. **摘要生成**：生成 200 字以内的内容摘要

**数据模型**：
```typescript
interface ExtractedContent {
  tags: string[];
  highlights: Highlight[];
  summary: string;
}

interface Highlight {
  text: string;
  context: string; // 前后各50字的上下文
}
```

## 四、API 设计

### 4.1 上传音频文件
```
POST /api/upload
Content-Type: multipart/form-data
Body: file=<音频文件>

Response: {
  audioId: string,
  fileName: string,
  duration: number,
  status: "pending"
}
```

### 4.2 提交转录任务
```
POST /api/transcribe
Content-Type: application/json
Body: { audioId: string }

Response: { taskId: string }
```

### 4.3 查询任务状态
```
GET /api/task/:taskId

Response: {
  status: "pending" | "processing" | "completed" | "failed",
  progress: number,  // 0-100
  result: {          // status=completed 时存在
    article: { id, title, content },
    extracted: { tags, highlights, summary }
  } | null,
  error: string | null
}
```

### 4.4 重新生成文章（基于已有转录）
```
POST /api/generate
Content-Type: application/json
Body: { transcriptId: string }

Response: {
  article: { id, title, content },
  extracted: { tags, highlights, summary }
}
```

### 4.5 导出 Markdown
```
GET /api/export/:articleId

Response: Markdown 文本（Content-Type: text/markdown）
Headers: Content-Disposition: attachment; filename="<编码后的标题>.md"
```

## 五、数据处理流程

### 5.1 完整流程
```
用户上传音频
    │
    ▼
┌─────────────┐
│  POST /api/upload  │
│  - 保存文件       │
│  - 创建 AudioFile │
└─────────────┘
    │
    ▼
用户点击"开始转录"
    │
    ▼
┌─────────────┐
│ POST /api/transcribe │
│ - 创建 Task         │
│ - 启动异步处理      │
└─────────────┘
    │
    ▼（异步）
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ Whisper.cpp │────▶│  保存转录   │────▶│ Minimax API │
│   转录音频   │     │  Transcript │     │  生成文章   │
└─────────────┘     └─────────────┘     └─────────────┘
                                              │
                                              ▼
                                       ┌─────────────┐
                                       │ 保存 Article │
                                       │ 更新 Task   │
                                       └─────────────┘
    │
    ▼（轮询）
GET /api/task/:taskId
    │
    ▼
显示结果 / 下载 Markdown
```

### 5.2 错误处理
- 上传失败：返回 500，显示错误信息
- 转录失败：Task 状态变为 failed，记录错误信息
- 生成失败：Task 状态变为 failed，记录错误信息
